# 核心产品逻辑变更说明
1. **交互降级与重构**：废弃原有的 "Ghost Text" 实时渲染。改为在侧边栏 UI 提供「当前沟通目标（意图）选择器」和「获取 AI 建议」按钮，由用户手动获取 2-3 条候选话术。
2. **重心转移 (从“我”到“他”)**：重点开发对「聊天对象」的画像分析（`contact.skill`），结合历史消息提取特征与用户手动标注，动态指导 AI 生成。

# 全新跨平台技术栈
我们将采用以下架构来实现桌面端应用，同时为未来向移动端迁移打好基础：
* **外壳与系统层**：**Tauri v2** (极简轻量，替代 Electron) + **Rust** (仅用于系统级探针和本地加密文件读写)。
* **前端视图与路由**：**Next.js (App Router)** + **React** + **TailwindCSS** + **Shadcn UI**。
* **AI 核心逻辑**：**Vercel AI SDK** (处理大模型对话状态与流式输出) + **LangChain.js / LlamaIndex.ts** (处理 RAG 和向量检索，直接在 Next.js Server Actions / API Routes 中运行)。
* **网络请求**：必须兼容本地网络代理工具（如 Surge，需考虑 Node.js 层的 `HTTP_PROXY`/`HTTPS_PROXY` 环境变量支持）。

# 阶段执行计划 (Step-by-Step)

请你严格按照以下四个阶段执行，每完成一个阶段，向我确认后再进入下一阶段。**注意：在初始阶段，探针能力通过 Mock 数据实现，优先跑通前后端与 AI 链路。**

## 阶段一：项目初始化与 UI 层构建 (Tauri + Next.js)
**目标**：搭建基础框架，利用 Shadcn UI 构建现代化的侧边栏交互界面。
**执行要求**：
1. 初始化 Tauri + Next.js (TypeScript) 项目结构。
2. 配置 Tauri 窗口属性：无边框 (frameless)、始终置顶 (always_on_top)、适当的侧边栏尺寸。
3. 使用 TailwindCSS 和 Shadcn UI 开发核心组件：
   - **Target 状态栏**：显示当前捕获到的聊天对象名称（本阶段先写死 Mock 数据，例如 "张总"）。
   - **Target 画像面板**：展示该对象的画像标签（如“强势”、“幽默”），允许用户动态增删标签。
   - **意图选择器 (Intent Selector)**：下拉框/选项卡，包含“日常闲聊”、“客户跟进”、“向上管理”等，支持自定义。
   - **对话区与建议卡片**：一个 "获取回应建议" 按钮，以及展示 Vercel AI SDK 返回的候选文本卡片（提供一键复制功能）。

## 阶段二：云端大脑与 RAG 重构 (Next.js API Routes + Vercel AI SDK)
**目标**：在前端服务端点中实现 BYOK（自带密钥）的大模型调用和聊天对象的画像 RAG 提取。
**执行要求**：
1. 引入并配置 `ai` (Vercel AI SDK) 和 `@ai-sdk/openai`，确保能兼容 DeepSeek / 豆包 等支持 OpenAI 协议的第三方模型。
2. **网络代理强调**：在实例化 OpenAI client 时，必须显式配置自定义的 `fetch` 或 `httpsAgent`，使其能够读取操作系统的代理环境变量（解决 Surge 等本地代理无法被 Node.js 原生接管的问题）。
3. 实现 RAG 逻辑：
   - 设计 `contact.skill` 数据结构（JSON 格式，包含 `manual_tags` 和 `distilled_traits`）。
   - 编写一个 Prompt Builder 聚合器，将【Mock的最新几条聊天记录】+【当前聊天对象画像】+【用户选择的意图】组装成 System Prompt，发送给大模型生成 3 条不同风格的回复建议。

## 阶段三：系统探针层开发 (Rust + macOS Accessibility)
**目标**：使用 Rust 编写 Tauri Command，调用 macOS 底层 API 抓取微信窗口信息。
**执行要求**：
1. 在 `src-tauri` 目录下，使用 Rust 及 macOS `core-graphics` / 辅助功能 API 编写系统探针。
2. 探针能力（严格只读）：
   - 寻找微信进程，获取当前激活的聊天窗口 Title（即聊天对象名称）。
   - 获取当前界面的最后几条聊天记录文本。
3. 将抓取到的数据暴露为 Tauri Command `#[tauri::command]`，供 Next.js 前端定时轮询或通过事件监听获取。*如果 API 获取失败，返回清晰的 Error，由前端 UI 友善提示“未捕获到聊天窗口”。*

## 阶段四：数据持久化与完善
**目标**：打通用户画像的本地保存，完善应用闭环。
**执行要求**：
1. 使用 Rust (Tauri API) 实现本地文件的读写。将生成的 `[contact_name].json` 画像文件安全地保存在本地 `AppData` 或应用专属目录下。
2. 联调前端修改标签 -> 触发 Rust 写入文件 -> 下次调用 AI 时读取最新文件的完整链路。
